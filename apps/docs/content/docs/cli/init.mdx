---
title: blodemd init
description: Scaffold a new documentation directory.
---

Create a starter documentation directory with a `docs.json` configuration file and an `index.mdx` page.

```bash
blodemd init [dir]
```

The `dir` argument defaults to `docs` if omitted.

## What it creates

Running `blodemd init` generates the following file tree:

<Tree>
  <Tree.Folder name="docs" defaultOpen>
    <Tree.File name="docs.json" />
    <Tree.File name="index.mdx" />
  </Tree.Folder>
</Tree>

### docs.json

The generated configuration uses the mint theme and a single "Getting Started" navigation group:

```json title="docs/docs.json"
{
  "$schema": "https://mintlify.com/docs.json",
  "name": "my-project",
  "theme": "mint",
  "colors": {
    "primary": "#0D9373"
  },
  "navigation": {
    "groups": [
      {
        "group": "Getting Started",
        "pages": ["index"]
      }
    ]
  }
}
```

### index.mdx

A minimal starter page with a title and placeholder content:

```mdx title="docs/index.mdx"
---
title: Welcome
---

Start writing your docs here.
```

## Options

<TypeTable
  type={{
    "[dir]": {

    },
  }}
/>

## Examples

<CodeGroup>
```bash title="Default directory"
blodemd init
```

```bash title="Custom directory"
blodemd init my-docs
```

</CodeGroup>

<Callout type="info">
  If the target directory already contains a `docs.json` or `index.mdx`, those
  files are not overwritten.
</Callout>

## Next steps

1. Open `docs.json` and set `name` to your project slug. This determines your `*.blode.md` subdomain.
2. Add more `.mdx` pages and update the `navigation` array.
3. Run `blodemd push` to deploy.
