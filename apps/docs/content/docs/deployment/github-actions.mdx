You can deploy your docs automatically on every push to your repository using GitHub Actions. The workflow runs `npx blodemd push` under the hood.

## Minimal workflow

This workflow deploys your docs whenever you push to the `main` branch:

```yaml title=".github/workflows/deploy-docs.yml"
name: Deploy docs

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npx --yes blodemd push
        env:
          BLODEMD_API_KEY: ${{ secrets.BLODEMD_API_KEY }}
```

<Callout type="info">
  The `--yes` flag tells npx to install the `blodemd` package without prompting.
</Callout>

## Production workflow with branch deploys

This workflow deploys the `main` branch to production and creates preview deployments for pull requests:

```yaml title=".github/workflows/deploy-docs.yml"
name: Deploy docs

on:
  push:
    branches: [main]
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - run: npx --yes blodemd push
        env:
          BLODEMD_API_KEY: ${{ secrets.BLODEMD_API_KEY }}
          BLODEMD_BRANCH: ${{ github.head_ref || github.ref_name }}
          BLODEMD_COMMIT_MESSAGE: ${{ github.event.head_commit.message }}
```

The `BLODEMD_BRANCH` value uses `github.head_ref` for pull requests (the source branch name) and falls back to `github.ref_name` for pushes.

## Using the composite action

Blode.md also provides a composite GitHub Action that wraps the CLI. You can use it instead of running `npx` directly:

```yaml title=".github/workflows/deploy-docs.yml"
name: Deploy docs

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: mblode/blodemd/packages/deploy-action@main
        with:
          api-key: ${{ secrets.BLODEMD_API_KEY }}
```

## Setting up the API key

You need a `BLODEMD_API_KEY` secret in your GitHub repository. Here is how to set it up:

<Steps>
  <Step title="Get your API key">
    Run `blodemd push` locally for the first time. When the project is created, the CLI prints an API key:

    ```
    Project my-project created
    API key for CI: bmd_abc123...
    ```

    You can also generate an API key from the Blode.md dashboard.

  </Step>
  <Step title="Add the secret to GitHub">
    1. Go to your repository on GitHub
    2. Navigate to **Settings** > **Secrets and variables** > **Actions**
    3. Click **New repository secret**
    4. Set the name to `BLODEMD_API_KEY` and paste your API key as the value
    5. Click **Add secret**
  </Step>
  <Step title="Reference the secret in your workflow">
    Use `${{ secrets.BLODEMD_API_KEY }}` in your workflow file as shown in the examples above.
  </Step>
</Steps>

<Callout type="warn">
  Never commit your API key directly in your workflow file. Always use GitHub
  Secrets.
</Callout>

## Specifying the docs directory

If your docs are not in the default location, pass the directory path as an argument or use the `directory` input:

<CodeGroup>
```yaml title="With npx"
- run: npx --yes blodemd push ./my-docs
  env:
    BLODEMD_API_KEY: ${{ secrets.BLODEMD_API_KEY }}
```

```yaml title="With composite action"
- uses: mblode/blodemd/packages/deploy-action@main
  with:
    api-key: ${{ secrets.BLODEMD_API_KEY }}
    directory: ./my-docs
```

</CodeGroup>

## Auto-detected environment variables

When running in GitHub Actions, the CLI automatically detects:

- **`GITHUB_REF_NAME`**: used as the branch name if `BLODEMD_BRANCH` is not set
- **Working directory**: the checkout root, where `docs.json` auto-detection starts
