title: Theming and branding

You control the visual identity of your docs through the `colors`, `logo`, `favicon`, `fonts`, and `appearance` fields in `docs.json`. This page covers each theming option in detail.

## Colors

The `colors` object defines your brand palette. Only `primary` is required -- all other values are derived automatically if omitted.

```json title="docs.json"
{
  "colors": {
    "primary": "#0D9373",
    "light": "#CFF6EE",
    "dark": "#0C3A33"
  }
}
```

  work well for most brands, so start with just `primary` and customize from

</Callout>

## Logo

The `logo` field sets the image displayed in the navbar. You can provide a single URL or separate light/dark variants.

<Tabs items={["Light and dark variants", "Single logo"]}>
  <Tab value="Light and dark variants">
    ```json title="docs.json"
    {
      "logo": {
        "light": "/images/logo-light.svg",
        "dark": "/images/logo-dark.svg"
      }
    }
    ```
  </Tab>
  <Tab value="Single logo">
    ```json title="docs.json"
    {
      "logo": "/images/logo.svg"
    }
    ```
  </Tab>
</Tabs>

<TypeTable
  type={{
    light: {
      description: "Logo image path or URL for light mode.",
      type: "string",
    },
    dark: {
      description: "Logo image path or URL for dark mode.",
      type: "string",
    },
    alt: {
      description: "Alt text for the logo image.",
      type: "string",
    },
    href: {
      description:
        "URL the logo links to when clicked. Defaults to your docs homepage.",
      type: "string",
    },
  }}
/>

<Callout type="info">
  Use SVG logos for the best quality at all screen sizes. Place logo files in
  your docs directory and reference them with a relative path like
  `/images/logo.svg`.
</Callout>

## Favicon

The `favicon` field sets the browser tab icon. Like logos, you can provide a single path or light/dark variants.

<CodeGroup>
```json title="Single favicon"
{
  "favicon": "/images/favicon.svg"
}
```

```json title="Light and dark variants"
{
  "favicon": {
    "light": "/images/favicon-light.svg",
    "dark": "/images/favicon-dark.svg"
  }
}
```

</CodeGroup>

## Fonts

The `fonts` object controls typography across your site. You can set separate families for headings, body text, and monospace code.

```json title="docs.json"
{
  "fonts": {
    "heading": "Inter",
    "body": "Inter",
    "mono": "JetBrains Mono",
    "provider": "google"
  }
}
```

<TypeTable
  type={{
    heading: {
      description: "Font family for headings (h1-h6).",
      type: "string",
    },
    body: {
      description: "Font family for body text and paragraphs.",
      type: "string",
    },
    mono: {
      description: "Font family for inline code and code blocks.",
      type: "string",
    },
    provider: {
      description:

      type: "string",
    },
  }}
/>

### Advanced font objects

For fine-grained control, you can use font objects instead of strings. This is useful for self-hosted fonts or when you need specific weights.

```json title="docs.json"
{
  "fonts": {
    "heading": {
      "family": "Cal Sans",
      "source": "/fonts/CalSans-SemiBold.woff2",
      "format": "woff2",
      "weight": 600
    },
    "body": {
      "family": "Inter",
      "source": "/fonts/Inter-Regular.woff2",
      "format": "woff2",
      "weight": 400
    }
  }
}
```

<TypeTable
  type={{
    family: {
      description: "CSS font-family name.",
      type: "string",
      required: true,
    },
    source: {
      description: "Path or URL to the font file.",
      type: "string",
    },
    format: {
      description: "Font file format hint for the browser.",
      type: "'woff' | 'woff2'",
    },
    weight: {
      description: "Numeric font weight (e.g., 400 for regular, 700 for bold).",
      type: "number",
    },
  }}
/>

### Using Google Fonts

Set `provider` to `google` and specify font family names. Blode.md automatically loads the fonts from Google Fonts CDN.

```json title="docs.json"
{
  "fonts": {
    "heading": "Poppins",
    "body": "Open Sans",
    "provider": "google"
  }
}
```

### Using a custom CSS URL

If you host your own font CSS (or use a service like Adobe Fonts), set `provider` to `custom` and provide the `cssUrl`.

```json title="docs.json"
{
  "fonts": {
    "heading": "MyCustomFont",
    "body": "MyCustomFont",
    "provider": "custom",
    "cssUrl": "https://fonts.example.com/my-custom-font.css"
  }
}
```

## Appearance

The `appearance` object controls dark/light mode behavior.

```json title="docs.json"
{
  "appearance": {
    "default": "system",
    "strict": false
  }
}
```

<TypeTable
  type={{
    default: {
      description: "Initial color mode when a visitor first loads the site.",
      type: "'dark' | 'light' | 'system'",
      default: "'system'",
    },
    strict: {
      description:
        "When `true`, locks the site to the `default` mode and hides the theme toggle. Useful for brand-specific designs.",
      type: "boolean",
      default: "false",
    },
  }}
/>

## Theme

The top-level `theme` field selects the overall visual template for your docs. Set it to a theme name like `mint`.

```json title="docs.json"
{
  "theme": "mint"
}
```

## Complete example

Here is a fully themed `docs.json`:

```json title="docs.json"
{
  "$schema": "https://mintlify.com/docs.json",
  "name": "my-project",
  "theme": "mint",
  "colors": {
    "primary": "#6366F1",
    "light": "#E0E7FF",
    "dark": "#312E81",
    "background": "#FAFAFA",
    "surface": "#F3F4F6"
  },
  "logo": {
    "light": "/images/logo-light.svg",
    "dark": "/images/logo-dark.svg"
  },
  "favicon": "/images/favicon.svg",
  "fonts": {
    "heading": "Inter",
    "body": "Inter",
    "mono": "JetBrains Mono",
    "provider": "google"
  },
  "appearance": {
    "default": "system"
  },
  "navigation": {
    "groups": [{ "group": "Guides", "pages": ["index"] }]
  }
}
```
