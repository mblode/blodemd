The multi-tenant template demonstrates a tenant-aware Next.js app that routes by hostname and serves multiple tenants from a single deployment.

<Callout type="info">
  Deploy the template directly from Vercel’s starter kit to explore a complete
  implementation.
</Callout>

## What is multi-tenancy?

Multi-tenancy means one application serves many tenants. Each tenant has a unique domain and configuration while sharing the same infrastructure.

## How it works

1. Requests hit the edge network
2. Middleware extracts the hostname
3. The request is rewritten to a tenant-aware route
4. Tenant-specific content is rendered

## Getting started

```bash title="Terminal"
git clone https://github.com/vercel/platforms
cd platforms
pnpm install
```

### Local development

Add tenant hostnames to your hosts file:

```text title="/etc/hosts"
127.0.0.1 tenant1.localhost
127.0.0.1 tenant2.localhost
127.0.0.1 custom.localhost
```

Start the dev server:

```bash title="Terminal"
pnpm dev
```

Visit different domains to see tenant-specific content.

## Core middleware

```ts title="middleware.ts"
import { type NextRequest, NextResponse } from "next/server";

export function middleware(request: NextRequest) {
  const hostname = request.headers.get("host") || "";

  if (request.nextUrl.pathname.startsWith("/domains")) {
    return NextResponse.next();
  }

  const target = `${request.nextUrl.href}domains/${hostname}`;
  const response = NextResponse.rewrite(target);
  response.headers.set("x-tenant-domain", hostname);
  return response;
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```
