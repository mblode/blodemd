This guide walks through the two domain strategies used by multi-tenant platforms: wildcard subdomains and tenant-owned custom domains.

<Video src="https://www.youtube.com/embed/vVYlCnNjEWA?si=FlkfRF7ktnKCS_VH" />

## Types of domains

### Wildcard domains (`*.acme.com`)

Use wildcard subdomains when you want tenant URLs like `tenant.acme.com`.

### Custom domains (`tenant.com`)

Use custom domains when tenants bring their own domain and point it at your platform.

## Wildcard domain setup

1. Point your domain to Vercel nameservers (`ns1.vercel-dns.com`, `ns2.vercel-dns.com`).
2. Add the apex domain in your Vercel project settings.
3. Add the wildcard domain entry: `.acme.com`.

Now any tenant subdomain (for example `docs.tenant1.acme.com`) routes to the same deployment and receives SSL automatically.

## Custom domain setup

### Step 1: Enable custom domains

Allow tenants to attach their own domain by provisioning it on your Vercel project and tracking verification status.

### Step 2: Add domains programmatically

```ts title="add-domain.ts"
import { VercelCore as Vercel } from "@vercel/sdk/core.js";
import { projectsAddProjectDomain } from "@vercel/sdk/funcs/projectsAddProjectDomain.js";

const vercel = new Vercel({
  bearerToken: process.env.VERCEL_TOKEN,
});

await projectsAddProjectDomain(vercel, {
  idOrName: "my-multi-tenant-app",
  teamId: "team_1234",
  requestBody: {
    name: "customacmesite.com",
  },
});
```

### Step 3: Verify domain ownership

If the domain is already in use on Vercel, request a TXT record and verify it.

```ts title="verify-domain.ts"
import { VercelCore as Vercel } from "@vercel/sdk/core.js";
import { projectsGetProjectDomain } from "@vercel/sdk/funcs/projectsGetProjectDomain.js";
import { projectsVerifyProjectDomain } from "@vercel/sdk/funcs/projectsVerifyProjectDomain.js";

const vercel = new Vercel({
  bearerToken: process.env.VERCEL_TOKEN,
});

const domain = "customacmesite.com";

const [domainResponse, verifyResponse] = await Promise.all([
  projectsGetProjectDomain(vercel, {
    idOrName: "my-multi-tenant-app",
    teamId: "team_1234",
    domain,
  }),
  projectsVerifyProjectDomain(vercel, {
    idOrName: "my-multi-tenant-app",
    teamId: "team_1234",
    domain,
  }),
]);

const { value: result } = verifyResponse;

if (!result?.verified) {
  console.log(`Domain verification required for ${domain}.`);
}
```

### Step 4: Configure redirects

Add both apex and `www` domains if needed, then set a redirect so traffic stays on the preferred domain. If a tenant has both a platform subdomain and custom domain, pick one as the canonical domain to avoid duplicate content.

### Step 5: Clean up domains

Remove custom domains when a tenant leaves your platform.

```ts title="remove-domain.ts"
import { VercelCore as Vercel } from "@vercel/sdk/core.js";
import { projectsRemoveProjectDomain } from "@vercel/sdk/funcs/projectsRemoveProjectDomain.js";
import { domainsDeleteDomain } from "@vercel/sdk/funcs/domainsDeleteDomain.js";

const vercel = new Vercel({
  bearerToken: process.env.VERCEL_TOKEN,
});

await Promise.all([
  projectsRemoveProjectDomain(vercel, {
    idOrName: "my-multi-tenant-app",
    teamId: "team_1234",
    domain: "customacmesite.com",
  }),
  domainsDeleteDomain(vercel, {
    domain: "customacmesite.com",
  }),
]);
```
