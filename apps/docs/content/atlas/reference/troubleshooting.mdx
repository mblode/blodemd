title: Troubleshooting
---

## DNS propagation delays

**Problem**: Domain not resolving after adding to the project.

**Solution**:

- DNS changes can take 24–48 hours to propagate
- Confirm nameservers are set correctly
- Clear local DNS cache when testing

## Domain verification failures

**Problem**: Verification fails after adding TXT record.

**Solution**:

- Wait 5–10 minutes for DNS propagation
- Confirm the TXT record matches exactly
- Remove duplicate TXT records
- Retry verification via SDK or dashboard

## Wildcard domain not working

**Problem**: Subdomains do not route to your app.

**Solution**:

- Ensure nameservers are `ns1.vercel-dns.com` and `ns2.vercel-dns.com`
- Verify the wildcard domain is added to the project
- Wait for DNS propagation

## SSL certificate not issued

**Problem**: Browser shows certificate errors.

**Solution**:

- Complete domain verification first
- Wait a few minutes for certificate issuance
- Check for CAA records blocking Let’s Encrypt

## Preview URL not resolving

**Problem**: Preview URLs do not work with custom domains.

**Solution**:

- Multi-tenant preview URLs require a custom preview suffix
- Use shorter branch names to avoid DNS label limits

## SEO duplicate content

**Problem**: Content is served on both subdomain and custom domain.

**Solution**:

- Set canonical URLs in metadata
- Redirect to a single primary domain
