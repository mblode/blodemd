---
title: Domain verification
---

Verification proves that a tenant owns the domain they want to connect to your platform.

## When verification is required

Verification is required when a domain is already configured on another Vercel project. Vercel returns a TXT record that must be added to the tenant’s DNS provider.

## Verification flow

1. Add the domain to your project.
2. Read the verification DNS record from the Vercel API.
3. Ask the tenant to add the TXT record.
4. Trigger verification with the SDK or dashboard.
5. Activate the domain once verified and SSL is issued.

## Example check

```ts title="verify-domain.ts"
import { Vercel } from "@vercel/sdk";

const vercel = new Vercel({
  bearerToken: process.env.VERCEL_TOKEN,
});

const result = await vercel.domains.getDomainConfig({
  domain: "example.com",
  teamId: "team_1234",
  slug: "my-team",
});

if (!result.verified) {
  console.log("Domain verification required.");
}
```

## Tips

- Wait a few minutes after adding the TXT record before retrying.
- Ensure the TXT value matches exactly (no trailing dots).
- Verify nameservers if you expect wildcard SSL.
